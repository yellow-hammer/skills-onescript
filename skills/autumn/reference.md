# Autumn — расширенная справка

## Аннотации ядра

- **&Желудь** — регистрация класса как компонента. Параметр — идентификатор (по умолчанию имя класса). Конструктор (ПриСозданииОбъекта) получает зависимости по типам/именам параметров.
- **&Верховный** — маркировка корневого компонента.
- **&Дуб** — класс-контейнер завязей, сам не создаётся как зависимость в произвольном месте.
- **&Завязь** — метод в Дубе, возвращающий экземпляр; используется для «ожелуживания» классов без правки их кода.

## Библиотека annotations

Autumn использует аннотации как объекты первого класса. Параметры аннотаций задаются в скобках: `&Желудь("MyId")`, `&КомандаПриложения(Имя = "cmd", Описание = "…")`. Поддерживаются мета-аннотации (агрегаторы других аннотаций). API описан в [документации Autumn — annotations](https://autumn-library.github.io/annotations).

## Настройки приложения

Файл `autumn-properties.json` рядом с точкой входа или в каталоге проекта позволяет задавать свойства для подсистем (например, cli: имя приложения, версия). Альтернатива — желудь с определённым интерфейсом (например, «ОписаниеКонсольногоПриложения» с методами ИмяПриложения, ВерсияПриложения).

## Модули экосистемы

- **autumn-collections** — прилепляемые коллекции.
- **autumn-cli** — консольные команды поверх cli.
- **autumn-logos** — логирование.
- **autumn-async**, **autumn-synchronized**, **autumn-event-publisher** — асинхронность, синхронизация, события.

Подключение через `#Использовать` и регистрация компонентов в том же каталоге классов, что и основное приложение, или через отдельные каталоги, указанные в точке входа.
